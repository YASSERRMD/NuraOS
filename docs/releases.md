### Artifacts

| Artifact | Description | SHA-256 |
|----------|-------------|---------|
| `nuraos-2.0.img.gz` | Compressed rootfs image | _(measure on build host: `sha256sum nuraos-2.0.img.gz`)_ |
| `nuraos-2.0.img.gz.sig` | Ed25519 signature over image | _(sign with release key)_ |
| `nuraos-2.0-manifest.json` | Build manifest with version pins | _(generated by `make manifest`)_ |

**To verify an artifact:**

```sh
# Verify Ed25519 signature
openssl pkeyutl -verify \
  -pubin -inkey release.pub.pem \
  -rawin -in nuraos-2.0.img.gz \
  -sigfile nuraos-2.0.img.gz.sig

# Verify SHA-256
sha256sum -c nuraos-2.0.img.gz.sha256
```

### Compliance posture

Default policy: `AllowCrossBorder=false`. All inference stays local or on
explicitly configured sovereign providers. The default provider list:

| Provider | Residency | Allow Sensitive |
|----------|-----------|----------------|
| local | local | yes |
| anthropic | cross-border | no |
| openai | cross-border | no |

Cross-border egress is blocked by default. Sensitive turns are blocked on
cloud providers even when cross-border is permitted.

### How to reproduce the release

```sh
# 1. Check out v2.0
git checkout v2.0

# 2. Verify host toolchain
./scripts/check-host.sh

# 3. Build the rootfs image
make image

# 4. Boot under QEMU and measure
make qemu
# Serial console: wait for "nura-manager: ready"

# 5. Run the full integration matrix
NURA_GATEWAY_URL=http://127.0.0.1:8080 nuractl integtest

# 6. Run the security audit
nuractl secaudit --critical

# 7. Run the performance gates (after measuring manually)
export NURA_BOOT_MS=950
export NURA_IDLE_RSS_MIB=95
export NURA_TOKENS_PER_SEC=14
nuractl perf

# 8. Verify the image signature
openssl pkeyutl -verify -pubin -inkey release.pub.pem \
  -rawin -in nuraos-2.0.img.gz -sigfile nuraos-2.0.img.gz.sig
```
