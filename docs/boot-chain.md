# NuraOS Boot Chain

This document describes the full boot sequence from power-on through the NuraOS
init process, covering both QEMU development paths and the production disk path.

---

## Boot modes

NuraOS supports two QEMU boot paths for development:

| Mode | Command | Bootloader |
|------|---------|------------|
| Direct kernel | `run-qemu.sh` | None (QEMU `-kernel` shortcut) |
| Bootloader disk | `run-qemu.sh --disk image/out/disk.img` | extlinux (syslinux) |

The bootloader disk mode exercises the same path as a real x86 machine.

---

## Disk image layout

`build-boot.sh` creates `image/out/disk.img`:

```
Offset 0        MBR (512 B) -- syslinux MBR code + DOS partition table
Offset 1 MiB   Partition 1 (FAT16, 64 MiB, bootable):
                  /bzImage           -- Linux kernel
                  /initramfs.cpio.gz -- packed initramfs
                  /extlinux.conf     -- boot menu configuration
                  /cmdline.conf      -- persisted extra kernel options
                  /ldlinux.sys       -- extlinux stage-2 loader
                  /menu.c32          -- text menu module (optional)
```

The `/data` persistent partition is a separate drive image (`data.img`) passed
via `-drive` and is not part of `disk.img`.

---

## Boot menu

`extlinux.conf` is generated by `scripts/boot-config.sh` and contains four
entries:

| Menu entry | Kernel cmdline addition | Default? |
|------------|------------------------|---------|
| NuraOS -- Slot A [active] | `nura.slot=a` | When active slot is A |
| NuraOS -- Slot B [active] | `nura.slot=b` | When active slot is B |
| NuraOS -- Recovery Mode | `nura.slot=<active> nura.recovery=1` | When `--recovery` passed |

The default entry is selected automatically from `/data/etc/active-slot` at
`build-boot.sh` time. Menu timeout is 5 seconds (50 tenths).

---

## Persisted cmdline options

Extra kernel options can be persisted across rebuilds in `image/out/boot/cmdline.conf`.
They are appended to every menu entry by `boot-config.sh`:

```
# image/out/boot/cmdline.conf
quiet
nura.debug=1
```

After editing this file, regenerate the config:

```sh
./scripts/boot-config.sh
```

---

## Kernel cmdline parameters

| Parameter | Values | Description |
|-----------|--------|-------------|
| `console` | `ttyS0,115200` | Serial console at 115200 baud |
| `panic` | `5` | Reboot 5 seconds after kernel panic |
| `loglevel` | `7` | Full kernel log verbosity |
| `nura.slot` | `a` or `b` | Rootfs A/B slot to mount (read by initramfs) |
| `nura.recovery` | `1` | Activate recovery mode in initramfs |
| `nura.debug` | `1` | Enable verbose init logging |

## Stage-by-stage boot sequence

### 1. BIOS / SeaBIOS (QEMU)

QEMU starts SeaBIOS which performs POST, then reads the MBR from the IDE disk
at LBA 0. The syslinux MBR code (440 bytes) is loaded and executed.

### 2. Syslinux MBR

The MBR reads the partition table, finds the bootable FAT16 partition at LBA
2048, and jumps to `ldlinux.sys` at the start of that partition.

### 3. extlinux / ldlinux.sys

`ldlinux.sys` is the syslinux stage-2 loader installed on the boot FAT partition.
It reads `extlinux.conf`, displays the boot menu (`menu.c32`), and waits for
the user to select an entry or for the timeout to expire.

### 4. Kernel + initramfs loading

extlinux loads `bzImage` and `initramfs.cpio.gz` from the FAT partition into
memory and passes the kernel cmdline from the selected menu entry.

### 5. Linux kernel initialisation

The kernel decompresses, initialises drivers (virtio-net, virtio-blk, IDE),
mounts the initramfs as the initial rootfs, and executes `/init`.

### 6. initramfs `/init`

`rootfs/init` (BusyBox sh) performs:

1. Mount `/proc`, `/sys`, `/dev`.
2. Parse `nura.slot` from `/proc/cmdline` (default: read `/data/etc/active-slot`).
3. If `nura.recovery=1`: start a BusyBox rescue shell.
4. Mount the selected rootfs slot (`/boot/rootfs-{a,b}.ext4`) at `/newroot`.
5. `switch_root /newroot /sbin/init` to hand off to the final init.

### 7. Final init (nura-manager)

The Rust supervisor (`nura-manager`) starts Go services (gateway, journal
forwarder), runs the AI inference loop, and enters its event loop.

---

## A/B slot selection

The active slot is determined in priority order:

1. `nura.slot=a|b` on the kernel cmdline (set by extlinux from `extlinux.conf`).
2. `/data/etc/active-slot` (written by `nuractl update apply` and `nuractl history rollback`).
3. Default: slot `a`.

When `nuractl update apply` succeeds, it flips `/data/etc/active-slot` and
updates `extlinux.conf` by re-running `boot-config.sh` so the next boot also
selects the new slot via the bootloader menu.

## Scripts reference

| Script | Purpose |
|--------|---------|
| `scripts/build-boot.sh` | Build `image/out/disk.img` with extlinux and boot files |
| `scripts/boot-config.sh` | Regenerate `extlinux.conf` and `cmdline.conf` |
| `scripts/run-qemu.sh` | Boot QEMU (direct-kernel or `--disk` bootloader path) |
| `scripts/build-image.sh` | Build kernel, initramfs, and data image |

### Build bootable disk

```sh
# After building kernel and initramfs:
./scripts/build-boot.sh

# Boot with extlinux:
./scripts/run-qemu.sh --disk image/out/disk.img
```

### Update boot config after slot change

```sh
# After nuractl update apply or nuractl history rollback:
./scripts/boot-config.sh
# Then rebuild the disk image:
./scripts/build-boot.sh
```

## Recovery boot

To force a recovery boot at the extlinux menu, either:

- Select "NuraOS -- Recovery Mode" from the interactive menu (press any key
  before the 5-second timeout to stop autoboot), or
- Pre-generate the config with recovery as default:

```sh
./scripts/boot-config.sh --recovery
./scripts/build-boot.sh
./scripts/run-qemu.sh --disk image/out/disk.img
```

In recovery mode, `rootfs/init` drops to a BusyBox shell before switching root,
allowing manual inspection and repair of `/data` or the rootfs slots.
